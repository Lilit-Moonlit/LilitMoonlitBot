# 📖 Інструкція: Як самостійно змінювати тексти в боті

Щоб ви могли редагувати повідомлення або написи на кнопках, не звертаючись щоразу за допомогою, я підготував цей гайд. Головне правило — **будьте обережні з пунктуацією та спеціальними символами**.

## 1. Де знаходяться тексти?

Більшість текстів розділені за ролями користувачів:

*   **Для клієнтів (пошук, запис, відгуки)**: `app/handlers/client.py`
*   **Для майстрів (профіль, пропозиції часу)**: `app/handlers/master.py`
*   **Назви кнопок меню**: `app/keyboards/main_kb.py`
*   **Каталог (категорії та послуги)**: `app/catalog.py`

## 2. Основні правила безпеки

### ❌ НЕ видаляйте лапки
Будь-який текст у Python завжди оточений лапками: `"текст"` або `'текст'`.
*   Правильно: `await message.answer("Привіт!")`
*   **НЕПРАВИЛЬНО**: `await message.answer(Привіт!)` — це викличе критичну помилку.

### ❌ Обережно з HTML-тегами
Бот використовує HTML для оформлення. Якщо ви відкрили тег `<b>`, його ОБОВ'ЯЗКОВО треба закрити `</b>`.
*   `<b>Жирний</b>`
*   `<i>Курсив</i>`
*   `<u>Підкреслений</u>`
*   `<a href='посилання'>Текст посилання</a>`

> [!WARNING]
> Якщо ви забудете закрити тег (наприклад, напишете `<b>Текст`), Telegram видасть помилку, і бот не зможе відправити це повідомлення.

### ❌ Символ `\n`
Це символ переносу рядка. Якщо ви бачите його всередині тексту, він означає, що наступне слово почнеться з нового рядка. Не видаляйте зворотний слеш `\`.

## 3. Редагування назв категорій (Дуже важливо!)

Назви категорій у `app/catalog.py` (наприклад, `"Нігті"`, `"Волосся"`) — це не просто текст. Вони використовуються як **ключі у базі даних**.

> [!CAUTION]
> Якщо ви зміните `"Нігті"` на `"Манікюр"` у файлі `catalog.py`, усі існуючі в базі послуги з категорією "Нігті" **зникнуть з меню**, бо бот не зможе їх знайти під новою назвою. Для таких змін завжди потрібна "міграція бази даних".

## 4. Як перевірити себе?

Після того, як ви щось змінили у файлі:
1.  Спробуйте запустити бота.
2.  Якщо ви допустили помилку в синтаксисі (забули лапку або дужку), бот просто не запуститься і напише в терміналі `SyntaxError`.
3.  Якщо бот запустився, перевірте саме ту кнопку або дію, де ви міняли текст.

## 5. Приклад безпечного редагування

Якщо ви хочете змінити вітання в `app/handlers/client.py`:

**Було:**
```python
text = f"Привіт, {message.from_user.first_name}! 🌸\nЯ твій помічник."
```

**Стало (безпечно):**
```python
text = f"Вітаємо, {message.from_user.first_name}! ✨\nМи допоможемо вам знайти майстра."
```
